### Acceleration Engine — new flagship surface (A1 → D3, complete)

The entire Acceleration Engine landed in one continuous overnight sequence, shipping the app's biggest new surface: a privacy-safe, evidence-cited workload-acceleration tool with deterministic mining, opt-in AI synthesis, and a durable skills library.

- **A1 — Foundation** (`packages/domain/src/models.ts`, `lib/audit.ts`) — `AccelerationSignal` / `AccelerationPlay` / `AccelerationPlayType` domain types; `acceleration_engine` audit event type wired into the exhaustive `AuditEventType` record.

- **B1 — Repetitive-sequence detector** (`packages/inference/src/accelerate.ts`) — pure `detectRepetitiveSequences()`: mines consecutive app-name n-grams (len 2–3, ≥ 3 recurrences) → AUTOMATE signals with `stableHash`-keyed `signal_id`, evidence citing app flow + counts + minutes (never window titles), confidence ramp from 0.5.

- **B2 — Time-sink detector** (`accelerate.ts`) — pure `detectTimeSinks()`: tallies reviewed `WorkBlock`s in 4 toolable categories (SQL/data modeling, recurring reporting, admin/coordination, dashboard work); emits TOOL signals only when ≥ 3 blocks AND non-deep time exists; minutes derived from `estimated_capacity_pct`.

- **B3 — Context-switch hotspot detector** (`accelerate.ts`) — pure `detectContextSwitchHotspots()`: walks consecutive session pairs, counts app-hops by local hour; emits TECHNIQUE signals for concentrated switch hours (≥ 3 switches AND top-1 hour beats mean × 1.5); evidence cites switch count + `hourWindowLabel` + refocus-minute estimate (Mark et al. CHI 2008 grounding).

- **B4 — Signal aggregator** (`accelerate.ts`, `hooks/useDerived.ts`) — `buildAccelerationSignals()` fans inputs across B1/B2/B3, dedupes by `signal_id`, collapses nested automate clusters (sub/superset `derived_from`), ranks by `minutes × confidence`, caps at 6 with early break.

- **C1 — Acceleration screen** (new `components/accelerate/AccelerationScreen.tsx`, `components/common/EvidenceDetails.tsx`, `lib/types.ts`, `lib/ui.ts`, `ScreenRouter.tsx`, `App.tsx`, `styles.css`) — new flagship `accelerate` screen added to the Agent section ("Ask | Accelerate" sub-tabs). Each Play card: humanized type chip, confidence figure with `title` + `.sr-only`, estimated time-saved metric, "Why this play?" evidence drill-down via shared `EvidenceDetails` (also now used by `BlockCard`). Header shows reclaimable minutes pill. Empty state via `EmptyState`.

- **C2 — Demo seeds** (`services/demoData.ts`) — 3 vivid evidence-cited Plays in `?demo=1&screen=accelerate` (~100 min/wk): AUTOMATE Hex→Looker→Teams handoff ×3 (37 min), TOOL SQL/data-modeling time-sink ×3 (42 min), TECHNIQUE 2pm context-switch hotspot (21 min). Added as session samples (zero capacity impact) + offset by trimming two reactive blocks to preserve the curated 22% reliable / 58% committed reconciliation.

- **C3 — Persist + audit play actions** (`App.tsx`, `localStore.ts`, `usePersistence.ts`, `lib/audit.ts`, `AccelerationScreen.tsx`, `AuditLogScreen.tsx`, `styles.css`, `demoData.ts`) — `dismissedPlayIds`/`savedPlayIds` arrays on `PersistedAppState`, all 4 hydration sites, `dismissPlay`/`savePlay`/`restoreDismissedPlays`/`unsavePlay` handlers. - **D1 — AI synthesis prompt + schema** (new `services/accelerationSchema.ts`, `services/accelerationPrompt.ts`) — strict-JSON `accelerationSchema` (`as const`, `additionalProperties:false`); `buildAccelerationPrompt` whitelists signal fields explicitly (never spreads, sends `derived_from_count` not ids) — privacy-by-construction.
