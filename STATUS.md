- Scope: edit `apps/desktop/src/` and `packages/` only.
- Gate: `npm run build` must pass before a task is marked done.

**Ordering** — the loop takes the first unchecked `- [ ]` task reading top-to-bottom

### Workplace Chat & Collaboration Signals
_New direction: turn workplace chat (Slack, Microsoft Teams, Webex) into reviewable workload signals. Mirror two proven patterns: the `gitLog.ts` burst-sessionizer (commits→sessions via `importRawEvents`) and the `calendarSource.ts` provider-registry + `loopSafe` flags + documented Rust follow-up. **Use a single generic `chat` `SourceType`** (generalize the reserved `slack` entry) rather than one per vendor — most orgs standardize on a single chat app, so the user picks their provider once and the workload signal is identical across Slack/Teams/Webex; the specific app rides as a `provider` field on event metadata + the `ChatSource` descriptor, not as a separate source type. Most of this is "data, not code."_
